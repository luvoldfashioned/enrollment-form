# 다단계 수강 신청 폼 프로젝트

React와 TypeScript를 기반으로 구축한 다단계 폼 프론트엔드 프로젝트입니다.
개인 및 단체 수강 신청에 필요한 3단계 폼을 구현했으며, 상태 관리 최적화와 유효성 검사 등 UX 개선에 집중했습니다.

## 프로젝트 개요

수강 신청 절차를 직관적인 스텝 인디케이터가 포함된 3단계 멀티스텝 형태로 제공하며, 조건부 유효성 검증 및 다양한 UX 개선 기능(임시 저장, 이탈 방지 등)을 구현한 프로젝트입니다.

## 기술 스택

- **Core**: React 19, TypeScript
- **Build**: Vite
- **Form**: React Hook Form, Zod
- **State/API**: TanStack Query, MSW
- **Styling**: Vanilla CSS Modules

## 실행 방법

- Node.js 18.x 이상

```bash
npm install
npm run dev
```

(MSW가 세팅되어 있어 별도의 API 서버 구동 없이 브라우저에서 즉시 동작합니다.)

## 요구사항 해석 및 가정

- **조건부 필드 데이터 정합성**: 단체 신청에서 개인 신청으로 전환할 때 기존에 입력한 단체 정보가 방치되어 데이터 무결성을 해치는 것을 방지하기 위해, 타입 변경 시 명시적 확인(Confirm) 후 관련 데이터를 즉시 초기화하도록 구현했습니다.
- **예외 상황 UX 처리**: 정원이 가득 찬 강의(`currentEnrollment >= maxCapacity`)는 사전에 비활성화하여 불필요한 폼 입력을 방지하고, 제출 시점에 발생하는 API 에러(`COURSE_FULL`, `DUPLICATE_ENROLLMENT`)는 하단 배너를 통해 사용자에게 즉각적이고 의미 있는 피드백을 전달하도록 해석했습니다.
- **스텝 간 데이터 유지 및 검증**: 각 스텝에서 이전 단계로 돌아갈 때 입력 데이터가 완벽히 유지되도록 최상위에서 통합된 폼 상태를 유지하며, 다음 단계로 넘어갈 때만 해당 스텝의 유효성을 검사하도록 가정했습니다. 또한 API 제출 실패 시에도 폼 상태가 완벽히 보존되어 사용자가 즉각적으로 내용을 수정하고 재시도할 수 있도록 설계했습니다.

## 설계 결정과 이유

1. **통합된 폼 상태와 Zod 판별 유니온 (Discriminated Union)**: 개인/단체 신청은 필수 필드가 완전히 다릅니다. React Hook Form으로 폼 상태를 통합 관리하면서, Zod의 `discriminatedUnion`을 통해 개인과 단체 타입을 명확히 분리하여 타입 안전성을 극대화했습니다.
2. **검증 로직과 UI 컴포넌트의 분리**: 유효성 검증 실패 시 사용자가 오류를 즉시 파악하고 수정할 수 있도록, 에러가 발생한 최초 필드로 시각적 포커스와 스크롤을 자동 이동시키는 로직을 `useFocusOnError` 커스텀 훅으로 분리했습니다.
3. **조건부 동적 명단 동기화**: 단체 신청 시 입력한 인원수(2~10명)에 맞춰 하단 참가자 명단(이름/이메일) 필드를 동적으로 렌더링하되, 인원수를 줄일 때 기존에 입력된 앞 번호 참가자 데이터가 유실되지 않도록 정교하게 동기화했습니다.
4. **디바운스 기반 임시 저장**: 폼 작성 도중 예기치 않은 새로고침이나 종료 시에도 데이터가 보존되도록 `localStorage` 자동 저장(Auto-save) 기능을 구현했습니다. 잦은 스토리지 접근으로 인한 병목을 막기 위해 300ms 디바운스(Debounce) 로직을 적용하여 성능을 최적화했습니다.

## 미구현 / 제약사항

- **이탈 방지 UI 커스텀 한계**: 입력 중 탭 닫기나 브라우저 뒤로가기 시 발생하는 `beforeunload` 이벤트는 최신 브라우저 보안 정책상 임의의 커스텀 디자인 UI를 띄울 수 없어 브라우저 기본 대화상자로 제한했습니다.
- **API 에러 모킹의 한계**: 실제 데이터베이스 연동이 없으므로, MSW(Mock Service Worker)를 활용해 1.5초의 로딩 지연과 정규식 기반 입력 검증, 그리고 일부 비즈니스 에러(`DUPLICATE_ENROLLMENT`)를 시뮬레이션하는 수준으로 구현되었습니다.

## AI 활용 범위

본 프로젝트는 Antigravity 2.0 환경(Gemini 3.5 Flash, Gemini 3.1 Pro, Claude Opus 4.6)에서 AI 코딩 어시스턴트를 활용하여 진행되었습니다.
- 초기 뼈대 구축 및 구현: 개발 기한 내에 완성도를 높이기 위해, 프로젝트의 보일러플레이트 세팅과 주요 기능 구현 코드의 상당 부분을 AI 모델을 활용해 작성했습니다.
- 학습과 문서화를 통한 코드 내재화: 제가 직접 작성하지 않은 코드를 제출하는 것에 그치지 않고, 낯선 상태 관리 로직(React Hook Form, Zod 판별 유니온 등)이 작동하는 원리를 완벽히 이해하기 위해 AI와 심층적인 질의응답을 진행했습니다. 이 과정에서 얻은 답변과 아키텍처 고민들을 정리하여 저만의 '기술 가이드 문서'로 남기며 코드를 학습하고 이해하는 데 집중했습니다.
